# reactive及其局限性

## reactive基础
- 创建响应式对象
- 深层响应性
- 与Vue2 Object.defineProperty的区别

## 局限性
- 解构时丢失响应性
- 替换整个对象导致的问题
- 对原始类型无效
- 对集合类型的特殊处理

## 相关API
- isReactive：检查对象是否由reactive创建
- shallowReactive：创建一个浅层响应式对象
- markRaw：标记一个对象，使其永远不会转换为代理
- toRaw：返回reactive或readonly代理的原始对象

## 解决局限性的策略
- 使用ref代替
- 使用toRefs保持响应性
- 使用扩展运算符的正确方式
